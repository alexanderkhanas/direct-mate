# FTP receiving endpoint (Torgsoft sync)

Minimal vsftpd container co-located with the existing DirectMate
production stack. Single-tenant for now: **luxespace**. Torgsoft
(Ukrainian desktop POS) uploads product CSV + images here; n8n
(separate workflow, not in this PR) polls the sync directory.

- Image: `fauria/vsftpd`
- Service name: `ftp` (in `infra/production/docker-compose.prod.yml`)
- Subdomain: `ftp.directmate.app`
- Single FTP user: `luxespace_trs` (chrooted)

Multi-tenant onboarding tooling, FTPS, and the n8n integration are
out of scope for this PR.

## One-time setup (operator)

All commands run on the Hetzner VPS unless noted.

### 1. DNS

Add an `A` record (manual, not in this repo):

```
ftp.directmate.app  A  <VPS public IPv4>
```

### 2. Hetzner firewall

Open these ports inbound (TCP):

- `21` — FTP control channel
- `21100-21110` — passive-mode data channels

The PASV port range is small on purpose: tighter firewall surface,
plenty of room for one polling client.

### 3. Host filesystem

Creates the bind-mount target with the right ownership. Must be root
because `/srv` and `chown` need it.

```bash
sudo bash scripts/ftp/setup-host.sh
```

This creates:

```
/srv/directmate-ftp/luxespace/
  ├── sync/    # TSGoods.trs lands here
  └── images/  # product photos (not consumed yet)
```

Owner is UID `14` GID `80` (the `ftp` user inside the
`fauria/vsftpd` container).

### 4. Configure .env

Edit `infra/production/.env` and set:

```
FTP_PASV_ADDRESS=<VPS public IPv4>
```

Find the IP with `curl -4 ifconfig.io` from the VPS.

Then generate the FTP password:

```bash
bash scripts/ftp/generate-credentials.sh
```

This appends `FTP_LUXESPACE_PASS=<random>` to `.env` and prints the
credentials block. **Save it in a password manager** — it is only
printed once.

### 5. Bring up the FTP service

```bash
cd infra/production
docker compose -f docker-compose.prod.yml up -d ftp
```

The other services keep running unchanged.

## Verification

After deploy, run these from your local machine (the operator's
laptop) — replace `<pass>` with the generated value.

### a) Container health

On the VPS:

```bash
cd infra/production
docker compose -f docker-compose.prod.yml ps ftp     # State should be Up (healthy)
docker compose -f docker-compose.prod.yml logs ftp   # No errors
```

### b) FTP connection (active mode)

From your local machine:

```bash
ftp ftp.directmate.app
# user: luxespace_trs
# pass: <pass>
ftp> ls
ftp> cd sync
ftp> put TSGoods-test.txt   # any small local file
ftp> ls                     # should show the upload
ftp> bye
```

On the VPS:

```bash
ls -la /srv/directmate-ftp/luxespace/sync/
# Should show TSGoods-test.txt, owner 14:80
```

### c) Passive mode (the one Torgsoft actually uses)

```bash
ftp -p ftp.directmate.app
# Same operations as above
```

If passive mode fails but active works, the firewall isn't
forwarding `21100-21110` — re-check Hetzner cloud firewall rules.

### d) Chroot isolation

```bash
ftp ftp.directmate.app
ftp> cd ..
ftp> pwd     # Must still be / (chroot working)
ftp> ls /etc # Empty or denied — must NOT show host /etc
```

### e) Wrong credentials rejected

```bash
ftp ftp.directmate.app
# Enter wrong password → "Login failed."
```

## Where files land on the host

| Inside FTP (chroot view) | Host path                                     |
| ------------------------ | --------------------------------------------- |
| `/`                      | `/srv/directmate-ftp/luxespace/`              |
| `/sync/`                 | `/srv/directmate-ftp/luxespace/sync/`         |
| `/images/`               | `/srv/directmate-ftp/luxespace/images/`       |

n8n's polling workflow (separate PR) will read from `/sync/`.

## Giving credentials to the Torgsoft operator

Torgsoft's "Об'єкт синхронізації" (sync object) UI fields:

| Torgsoft field          | Value                                |
| ----------------------- | ------------------------------------ |
| Адреса (host)           | `ftp.directmate.app`                 |
| Порт                    | `21`                                 |
| Користувач (login)      | `luxespace_trs`                      |
| Пароль                  | (generated by `generate-credentials.sh`) |
| Шлях до файлів синхро.  | `/sync`                              |
| Шлях до зображень       | `/images`                            |
| Режим передачі          | passive                              |

Send these via a secure channel (password manager share, Signal —
not plain email/Telegram).

## Troubleshooting

- **Login works but `ls` hangs**: passive ports `21100-21110` aren't
  reachable. Check Hetzner firewall + that `PASV_ADDRESS` matches
  the public IP.
- **`530 Login failed`**: password mismatch between `.env` and what
  you handed to the operator. Re-run `generate-credentials.sh` to
  see the current value (it doesn't regenerate if already set).
- **Files appear on host but n8n doesn't see them**: out of scope
  for this PR — debug the n8n workflow.
- **Container restart loop**: `docker compose logs ftp` — likely a
  missing env var (`FTP_PASV_ADDRESS` empty) or the bind-mount path
  doesn't exist (run `setup-host.sh`).

## Why no FTPS?

Torgsoft only supports plain FTP. Mitigations for v1:

- Strong randomly-generated password (32 chars).
- Firewall locked to FTP ports only.
- Chrooted user with no shell.
- No backend integration yet — files just sit on disk for n8n
  review-and-dump-to-Sheets.

If FTPS becomes required (or a non-Torgsoft client lands), revisit
by enabling vsftpd's TLS support and minting a cert via the
existing certbot service.
